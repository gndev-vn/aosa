## 3. Encryption Model (Zero-Trust)

### 3.1 Key Derivation

```
User PIN (UTF-8) + Random Salt (32 bytes)
        в”Ӯ
        в–ј
   Argon2id(id: 3, mem: 64MB, time: 3, par: 4)
        в”Ӯ
        в–ј
   AES-256-GCM Key (256 bits)
```

- Salt is stored alongside the encrypted blob (not secret вҖ” it's a public input to KDF)
- Argon2id parameters make brute-force infeasible even if salt is known
- Same PIN + same salt вҶ’ same key (deterministic for cross-device)
- When user changes PIN, all salts are regenerated and data re-encrypted

### 3.2 Encryption Format (per OTP record)

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ version: 1  (1 byte)                             в”Ӯ
в”Ӯ salt:      32 bytes                              в”Ӯ
в”Ӯ nonce:     12 bytes (AES-GCM IV)                 в”Ӯ
в”Ӯ ciphertext: variable length (encrypted payload)  в”Ӯ
в”Ӯ tag:       16 bytes (GCM auth tag)               в”Ӯ
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

### 3.3 Server Storage (opaque)

The server stores only:
```
{
  "id": "uuid",
  "encrypted_blob": "base64(version + salt + nonce + ciphertext + tag)",
  "version": 5,
  "created_at": "2026-06-13T00:00:00Z",
  "updated_at": "2026-06-13T00:00:00Z",
  "deleted_at": null | "timestamp"
}
```

The server **cannot** read the OTP secret. It is an opaque encrypted blob store.
